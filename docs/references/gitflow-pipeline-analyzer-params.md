# gitflow-cli pipeline analyzer 完整参考

> 本文档为 `gitflow-pipeline-analyzer` skill 的外部化引用。

## 流水线相关命令速查

```bash
gitflow-cli pipeline report --branch <BRANCH> --days <N>
gitflow-cli pipeline status --branch <BRANCH>
gitflow-cli pipeline jobs --pipeline-id <ID>
gitflow-cli pipeline logs --pipeline-id <ID>
```

## 默认分析参数

| 参数 | 默认 | 说明 |
|------|------|------|
| `--branch` | `git branch --show-current` | 目标分支 |
| `--days` | 7 | 分析天数范围（7/14/30） |

## 质量等级阈值

| 成功率 | 等级 |
|--------|------|
| ≥ 95% | 🟢 健康 |
| 80% ~ 94% | 🟡 关注 |
| < 80% | 🔴 告警 |

## 失败模式分类

| 失败类型 | 典型根因 |
|----------|----------|
| build 失败 | 编译错误、依赖问题 |
| test 失败 | 单元测试失败、集成测试超时 |
| lint 失败 | 代码格式、静态分析警告 |
| deploy 失败 | 环境配置、权限问题 |
| timeout | 资源不足、死锁 |

## 改进建议优先级

| 优先级 | 条件 |
|--------|------|
| 🔴 紧急 | 成功率 < 80% 或 build 持续失败 |
| 🟠 高 | 成功率 80%~94% 或存在 flaky test |
| 🟡 中 | 耗时增长 > 20% 或 lint 频繁失败 |
| 🟢 低 | 成功率 ≥ 95% 但仍有优化空间 |
